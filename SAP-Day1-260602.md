## 1. 핵심 개념

### IT System 정리
- OLTP (OnLine Transaction Processing) 거래 기록
  - System of Records
- OLAP (OnLine Analytic Processing) 거래 분석
  - 다 의사 결정을 위함... 
- DW (Data Warehouse)
- Data Lakehouse
- Data Fabric

### Enterprise Application 기업용 어플리케이션

- ERP + Extended Soln. (CRM, SCM, et cetera)

- ERP: 전사적 자원관리
  - Four Walls (기업 내부 Boundary)
- CRM: 고객관계관리 (예: Salesforce)
- SCM: 공급망관리
  - 공급망: 공급업체, 기업, 운송업체, 고객 등 물류의 흐름 전체
  - SCM의 종류
    - SCP (Planning): 공급망계획
    - SCE (Execution): 공급망실행
- SRM: 공급업체관계관리
- OMS: 주문관리시스템
- HR: PeopleSoft
- SaaS (Software-as-a-Service) vs. Hardware Server

### PLM 계열

- CAD/CAM
  - Computer Aided Design/Manufacturing의 준말
- PDM (Product Data Mgmt.)
- PLM (Lifecycle): 제품생애주기관리
  - 제품생애주기: 제품의 아이디어 발굴, 설계, 양산, 변경관리, 폐기에 이르는 전체 구간
- CAM은 CAD를 포함하는 개념.

### 통합 방식

- EAI: Interface 통합 (Process Integration)
- MDM: 기준정보 통합 (Information Integration)
- Portal: UX 통합 (Ppl. Integration)
  - SSO (Single Sign-On)

### Info. Silo vs. ERP

- Info. Silo: 개별적 기록
- ERP (Enterprise Resource Planning): 통합 기록, Single Source of Truth

## 2. MRP / ERP 발전 과정

### MRP 계열

- MRP (Material Requirement Planning): 자재소요량계획. 하나의 완제품을 만들기 위해 어떤 자재들이 얼만큼 필요한가?
- MRP-II: MRP + CRP (Capacity Resource Planning, 생산역량계획)

### SAP 개요

- System, Application, Program in data processing (SAP)
- 기존 H/W에 딸려서 오던 S/W (SI: System Integration; SM: System Mgmt.) vs. 패키지형 S/W
- 예: J. D. Edwards

### SAP R/ 표기

- SAP R/1에서 R의 의미: Real-Time
- R/3: (Client/Server Architecture의) Tier 갯수

### 과거 기업별 OS 및 DB

- IBM: AIX / DB2
- Hewlett-Packard: HP-UX / DB 없음
- MS: Windows Server / MS-SQL
- Sun Microsystems: Solaris
- Oracle: OS 없음 / RDBMS (Relational)

### Oracle Applications

- SAP-Oracle 묶음판매 계약 (뽀찌 Contract)
- Oracle 버전의 ERP: Oracle Applications (Internet 도입에 적극적)
- 사용 기업 예: KT, 대한항공, LG전자, 효성, 오뚜기
- 부족한 기능은 타 ERP 인수로 보충 (예: PeopleSoft)
- Project Fusion: 난전 100개를 백화점으로 통합 시도 → 통합 실패 후 도태

### HANA

- SAP의 DB
- StarSchema (서울대 차모 교수가 개발)
- 억지 역두문자어

## 5. 확장 솔루션 및 E2E 프로세스

### 솔루션 구성

- ERP + 확장 Soln. (Extended Soln.): CRM, SRM 등
- ERP + LoB (Line-of-Business): 특정 업무 지원 솔루션

### E2E (End-to-End) Process vs. 기존 벌집모델

- Buy: 구매
- Supply: 공급
- Design: 설계
- Make: 생산
- Maintain: 유지관리
- Sell: 판매
- Service: 고객서비스

### E2E 프로세스 명칭

- Source to Pay: 조달에서 대금지급까지
- Design to Operate: 설계에서 생산까지
- Lead to Cash: 잠재고객에서 대금수금까지
- Request to Service: 서비스 요청부터 제공까지
- Hire to Retire: 채용에서 은퇴까지 (인사)
- Record to Report: 장부기입에서 보고서까지
